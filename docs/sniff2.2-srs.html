Sniff v2.2

System Requirements

December 09, 2004

1. Introduction:

Sniff v2.2 is a network monitoring tool intended to be used on platforms that run POSIX Operating Systems (Linux/BSD/Unix). It has been developed on Redhat Linux 9.0 platform using the C programming language. It makes use of external libraries for packet capturing, user interface and Unix OS features for creating shared memory buffer and semaphores. Statically linked binaries are available with the release which do not need the packet capturing libraries for execution. This document describes the system requirements for compilation, installation and execution of the software. Kindly go through the Design Document and the User Documentation also.


2. External Libraries:

2.1 PCAP Library:

The software makes use of the PCAP library, which is available for all POSIX Operating Systems (Linux/BSD/Unix). Statically linked binaries, which do not need the PCAP library for their execution, are provided with the release. However, the library is necessary for compiling the source.

Libpcap v0.7.2 (and earlier versions) had certain vulnerabilities (w.r.t. certain ISAKMP packets) which used to cause packet capture buffer overflow and subsequent crashing of the entire network interface. Luckily, the developers of Libpcap have fixed this flaw in the latest release of Libpcap v0.8.3. Users of Sniff are advised to use Libpcap v0.8.3 (and onwards) while compiling the source. The statically linked binaries provided with Sniff v2.2 use the benign Libpcap-v0.8.3.

2.2 Unix Library for Shared Memory and Semaphore Management:

The software requires Unix Memory Management (MM) and Interprocess Communication (IPC) libraries for compilation and execution. They are used for implementing shared memory buffer and semaphore operations respectively.

2.3 NCurses Library:

The software makes use of NCurses library for the summary display when the tool is being run. The statically linked binaries do not need the library. But compilation of the source requires the library to be installed in the system.


3. Operating System Requirements:

The statically linked binaries require platforms running POSIX operating systems for successful execution. Also, the processes need root privileges for accessing the network interfaces. Compilation of the source has an additional requirement of the availability of the required libraries.


4. Compiler Requirements:

4.1 Source Compilation:

The tool has been programmed in the C programming language. The source has been successfully compiled using GCC ver3.x (GNU's C Compiler).

4.2 Webpage Interface Interpretation:

The webpage interface supported by the system requires Perl to interpret the CGI script. Also, the web server needs to be configured to run the CGI script. The script needs to be installed in the appropriate directory.
